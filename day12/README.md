# Агент сбора реквизитов с трёхслойной моделью памяти

CLI-агент для получения реквизитов компании через OpenAI-совместимый API.

## Архитектура памяти

Агент использует **три слоя памяти**, каждый хранится в отдельном JSON-файле:

```
┌─────────────────────────────────────────────────────────────┐
│                    ДОЛГОВРЕМЕННАЯ ПАМЯТЬ                     │
│                  (memory_long_term.json)                     │
│  • Профиль пользователя (имя, роль, компания)               │
│  • Предпочтения (формат общения, краткость)                 │
│  • История успешно собранных компаний                       │
│  • Паттерны поведения                                       │
│  Время жизни: месяцы/годы                                   │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ редко обновляется
                              │
┌─────────────────────────────────────────────────────────────┐
│                      РАБОЧАЯ ПАМЯТЬ                          │
│                   (memory_working.json)                      │
│  • Какие реквизиты уже собраны                              │
│  • Какие поля ещё нужны                                     │
│  • Текущий статус задачи                                    │
│  • Блокеры                                                  │
│  Время жизни: одна задача                                   │
└─────────────────────────────────────────────────────────────┘
                              ▲
                              │ обновляется после каждого сообщения
                              │
┌─────────────────────────────────────────────────────────────┐
│                   КРАТКОСРОЧНАЯ ПАМЯТЬ                       │
│                 (memory_short_term.json)                     │
│  • Сырая история диалога                                    │
│  • Все сообщения user/assistant с timestamp                 │
│  Время жизни: текущая сессия                                │
└─────────────────────────────────────────────────────────────┘
```

## Как это работает

### 1. Каждое сообщение пользователя:

```
[Сообщение пользователя]
        │
        ▼
┌─────────────────────┐
│ Сохранить в         │
│ краткосрочную       │
│ память              │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ Собрать контекст:   │
│ • working memory    │
│ • long-term memory  │
│ • последние N       │
│   сообщений         │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ LLM #1: Основная    │  ← system prompt с контекстом памяти
│ модель (gpt-4o)     │
└─────────────────────┘
        │
        ▼
   [Ответ пользователю]
        │
        ▼
┌─────────────────────┐
│ LLM #2: Экстрактор  │  ← анализирует последние сообщения
│ (gpt-4o-mini)       │
└─────────────────────┘
        │
        ▼
┌─────────────────────┐
│ Обновить:           │
│ • рабочую память    │
│ • долговременную    │
│   (если есть что)   │
└─────────────────────┘
```

### 2. Экстрактор памяти

Отдельный вызов дешёвой модели, который:
- Извлекает структурированные данные из диалога
- Определяет, какие реквизиты уже собраны
- Обновляет список недостающих полей
- Ловит факты о пользователе для долговременной памяти

**Один вызов → распределение по двум слоям** (рабочая + долговременная).

## Установка и запуск

```bash
export OPENAI_BASE_URL="https://api.openai.com/v1"
export OPENAI_API_KEY="sk-..."

python3 main.py
```

### Параметры

```bash
python3 main.py --help

# Основные:
--model gpt-4o              # Основная модель
--extractor_model gpt-4o-mini  # Модель для экстрактора
--json_format               # Финальный ответ в JSON
--profile user.md           # Файл с профилем пользователя

# Память:
--short_term_path memory_short_term.json
--working_path memory_working.json
--long_term_path memory_long_term.json
--context_messages 10       # Сколько сообщений в контекст
```

## Профили пользователя

Профиль — это `.md` файл с предпочтениями по стилю, формату и ограничениям. Передаётся через `--profile` и вставляется в начало system prompt.

```bash
# Сухой деловой стиль
python3 main.py --profile example_profile.md

# Дружелюбный с эмодзи
python3 main.py --profile friendly_profile.md
```

### Пример: `example_profile.md` (лаконичный)

```markdown
## Стиль общения
- Отвечай кратко и по делу
- Не используй эмодзи
- Обращайся на "ты"

## Формат ответов
- Не добавляй вводные фразы типа "Конечно!", "С удовольствием!"
- Задавай только один вопрос за раз
```

### Пример: `friendly_profile.md` (дружелюбный)

```markdown
## Стиль общения
- Будь максимально дружелюбным и приветливым 🌟
- Используй эмодзи в каждом сообщении
- Обращайся на "Вы", называй "уважаемый клиент"

## Формат ответов
- Начинай с позитивной фразы ("Отлично!", "Замечательно!")
- Благодари за каждую предоставленную информацию
```

Профили позволяют одному агенту работать в разных «персонах» без изменения кода.

## Команды в чате

| Команда       | Описание                                    |
|---------------|---------------------------------------------|
| `/memory`     | Показать состояние всех слоёв памяти        |
| `/short`      | Показать краткосрочную память (история)     |
| `/working`    | Показать рабочую память (данные задачи)     |
| `/long`       | Показать долговременную память (профиль)    |
| `/reset`      | Сбросить рабочую память (новая задача)      |
| `/newsession` | Очистить краткосрочную память (новая сессия)|
| `/help`       | Справка по командам                         |

## Структура файлов

```
.
├── main.py                  # CLI-интерфейс
├── agent.py                 # Основной агент с интеграцией памяти
├── memory_manager.py        # Менеджер трёхслойной памяти
├── memory_extractor.py      # LLM-экстрактор для обновления памяти
├── example_profile.md       # Профиль: лаконичный стиль
├── friendly_profile.md      # Профиль: дружелюбный стиль
├── memory_short_term.json   # Краткосрочная память
├── memory_working.json      # Рабочая память
└── memory_long_term.json    # Долговременная память
```

## Ключевые решения

1. **Явное управление памятью** — агент (через экстрактор) решает, что сохранять в какой слой
2. **Разделение хранения** — каждый слой в своём файле
3. **Экономия токенов** — экстрактор использует дешёвую модель
4. **Персистентность** — состояние сохраняется между запусками
