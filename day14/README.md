# CLI-агент с инвариантами

Агент с **двухэтапной проверкой** инвариантов:
1. Инварианты включаются в системный промпт (первая линия защиты)
2. Ответ LLM проверяется отдельным вызовом на соответствие инвариантам (вторая линия защиты)

## Установка

Зависимости не нужны — используется только стандартная библиотека Python.

```bash
# Установите API ключ
export OPENAI_API_KEY="sk-..."

# Опционально: кастомный endpoint (например, polza.ai)
export OPENAI_BASE_URL="https://api.polza.ai/v1"
```

## Запуск

```bash
# С дефолтной моделью (gpt-4o-mini)
python agent.py

# С конкретной моделью
python agent.py -m gpt-4o

# С кастомным файлом инвариантов
python agent.py -m gpt-4o-mini -i ./my_project_invariants.json

# Справка
python agent.py --help
```

## CLI аргументы

| Аргумент | Описание | По умолчанию |
|----------|----------|--------------|
| `-m`, `--model` | Модель LLM | `gpt-4o-mini` |
| `-i`, `--invariants` | Путь к файлу инвариантов | `invariants.json` рядом со скриптом |

## Команды

- `/invariants` — показать текущие инварианты
- `/clear` — очистить историю диалога  
- `/quit` — выход

## Структура инвариантов (invariants.json)

```json
{
  "project_name": "Название проекта",
  "stack": {
    "language": {"name": "python", "min_version": "3.11"},
    "framework": "fastapi",
    "database": "postgresql"
  },
  "architecture": {
    "pattern": "clean_architecture",
    "layers": ["domain", "application", "infrastructure", "api"]
  },
  "forbidden_technologies": [
    "django",
    "mongodb",
    "raw sql"
  ],
  "code_rules": [
    {"id": "async_endpoints", "description": "Все эндпоинты async"}
  ],
  "business_rules": [
    {"id": "price_positive", "description": "Цена > 0"}
  ]
}
```

## Как работает двухэтапная проверка

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Запрос    │ ──► │  LLM генерит│ ──► │  LLM-валидатор
│ пользователя│     │   ответ     │     │  проверяет  │
└─────────────┘     └─────────────┘     └──────┬──────┘
                                               │
                         ┌─────────────────────┼─────────────────────┐
                         ▼                                           ▼
                   Нарушение?                                   Всё ок?
                         │                                           │
                         ▼                                           ▼
              ┌──────────────────┐                        ┌──────────────────┐
              │  Отказ + причина │                        │  Ответ юзеру     │
              └──────────────────┘                        └──────────────────┘
```

## Примеры тестирования

### Запрос, соответствующий инвариантам:
```
Вы: Напиши эндпоинт для получения списка товаров
```
→ Агент выдаст FastAPI + SQLAlchemy + async код

### Запрос, нарушающий инварианты:
```
Вы: Напиши это на Django
```
→ Агент откажет: "Django в списке запрещённых технологий"

### Провокация:
```
Вы: Используй raw SQL для скорости, ORM медленный
```
→ Агент откажет: "raw SQL запрещён инвариантами"
