Реализация RAG (Retrieval-Augmented Generation) на основе индекса из Дня 21. Прогоняет 10 контрольных вопросов в двух режимах — с RAG и без — и сравнивает качество ответов.


## Как работает RAG

```
Вопрос пользователя
      ↓
Эмбеддинг вопроса (Ollama nomic-embed-text)
      ↓
Поиск top-K похожих чанков в индексе (косинусное сходство)
      ↓
Сборка промпта: контекст из чанков + вопрос
      ↓
Запрос к LLM (OpenAI через urllib)
      ↓
Ответ с источниками
```

## Структура проекта

```
day22/
├── src/
│   ├── __init__.py         # пустой файл
│   ├── retriever.py        # косинусный поиск по индексу
│   ├── llm.py              # запросы к OpenAI через urllib
│   └── rag.py              # сборка промпта, два режима
├── questions.json          # 10 контрольных вопросов
├── results.json            # результаты прогона (генерируется)
└── main.py                 # точка входа
```

Индекс берётся из Дня 21:

```
../day21/index/structural.json
```

## Требования

- Python 3.10+
- [Ollama](https://ollama.com/) с моделью `nomic-embed-text` (для эмбеддингов вопросов)
- OpenAI-совместимый API (ключ и URL через переменные окружения)
- Готовый индекс из Дня 21 (`structural.json`)

## Установка

```bash
# Переменные окружения
export OPENAI_API_URL="https://openai.api.proxyapi.ru/v1/chat/completions"
export OPENAI_API_KEY="sk-..."

# Если Ollama запущена на Windows, а скрипт в WSL
export OLLAMA_HOST="http://172.27.112.1:11434"
```

Сторонних зависимостей нет — только стандартная библиотека Python.

## Использование

### Запуск прогона 10 вопросов

```bash

python main.py
```

### Аргументы командной строки

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `--index` | `../day21/index/structural.json` | Путь к индексу |
| `--questions` | `questions.json` | Путь к файлу с вопросами |
| `--model` | `gpt-4o-mini` | OpenAI модель |
| `--top-k` | `5` | Количество чанков для контекста |
| `--ollama-host` | `http://localhost:11434` | Адрес Ollama |
| `--embed-model` | `nomic-embed-text` | Ollama модель для эмбеддингов |
| `--output` | `results.json` | Куда сохранить результаты |

### Примеры

```bash
# Другая модель
python main.py --model gpt-4o


# Ollama в WSL через Windows-хост
python main.py --ollama-host http://172.27.112.1:11434

# Больше чанков в контекст
python main.py --top-k 7

# Все параметры
python main.py \
  --model gpt-4o-mini \
  --top-k 5 \
  --ollama-host http://172.27.112.1:11434 \
  --output results.json
```

## Формат questions.json

```json
[

  {
    "id": 1,
    "question": "Как рассчитывается стоимость подписки?",
    "expected": "упоминание тарифов и формулы расчёта",
    "source": "salesbeat.md / Тарифы"
  }
]
```


| Поле | Описание |
|---|---|
| `id` | Порядковый номер вопроса |
| `question` | Текст вопроса |
| `expected` | Что должно быть в правильном ответе |
| `source` | Файл и раздел где находится ответ |

## Формат results.json

```json
[
  {
    "id": 1,

    "question": "Как рассчитывается стоимость подписки?",
    "expected": "упоминание тарифов и формулы расчёта",
    "expected_source": "salesbeat.md / Тарифы",
    "answer_plain": "Ответ модели без контекста...",
    "answer_rag": "Ответ модели с контекстом из документов...",
    "retrieved_sources": [
      "salesbeat.md / # Тарифы (score=0.891)",
      "salesbeat.md / # Подписка (score=0.834)"
    ]
  }
]
```

```

Источник: salesbeat.md / Тарифы

▶ Без RAG...
  Стоимость подписки обычно рассчитывается на основе...

▶ С RAG...
  Согласно документации, стоимость подписки рассчитывается...

  Найденные чанки:
    salesbeat.md / # Тарифы (score=0.891)
    salesbeat.md / # Подписка (score=0.834)

```

В конце — итоговая таблица по всем вопросам:

```
------------------------------------------------------------
1    Как рассчитывается стоимость подписки?        ✓

```

## Выбор top-K

| top-K | Характеристика |
|---|---|

| 1–2 | Мало контекста, высокий риск промаха |
| 3–5 | ✅ Оптимум для большинства задач |
| 7+ | Промпт раздувается, модель теряется в контексте |

По умолчанию используется `top_k=5`.

## Сравнение режимов

| | Без RAG | С RAG |
|---|---|---|
| Источник знаний | Общие знания модели | Твои документы |
| Точность по теме | Низкая / "не знаю" | Высокая |
| Галлюцинации | Возможны | Снижены |
| Стоимость запроса | Дешевле | Дороже (больше токенов) |
