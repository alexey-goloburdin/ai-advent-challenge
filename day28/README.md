RAG-пайплайн с LLM-реранкингом и структурированным выводом. Модель обязана возвращать ответ, список источников с номерами чанков и дословные цитаты из контекста. Поддерживает режим "не знаю" при слабой релевантности найденных чанков.

## Пайплайн

```
Вопрос
  ↓
Эмбеддинг вопроса (Ollama nomic-embed-text)
  ↓
Cosine search → top-20 кандидатов

  ↓
LLM reranker → оценка каждого чанка 0–10
  ↓
top-5 по rerank score
  ↓  [если лучший score < dont-know-threshold → "не знаю"]

LLM генерирует структурированный JSON-ответ
  ↓
{ answer, sources, quotes }
```

## Структура проекта

```
day24/
├── src/
│   ├── __init__.py         # пустой
│   ├── llm.py              # запросы к LM Studio через urllib

│   ├── reranker.py         # LLM реранкер (оценка 0-10)
│   └── rag.py              # пайплайн + структурированный вывод
├── questions.json          # 10 контрольных вопросов
├── results.json            # результаты прогона (генерируется)
└── main.py                 # точка входа
```

Индекс берётся из Дня 21:

```
../day21/index/structural.json
```

## Требования

- Python 3.10+
- [LM Studio](https://lmstudio.ai/) с двумя загруженными моделями:
  - `qwen/qwen3-5b` (или совместимая) — для генерации и реранкинга
  - `nomic-ai/nomic-embed-text-v1.5-GGUF` — для эмбеддингов
- Готовый индекс из Дня 21 (`structural.json`)

Сторонних зависимостей нет — только стандартная библиотека Python. API-ключ не требуется.


## Использование


### Базовый запуск

```bash
python main.py --lm-studio-url http://localhost:1234
```

### Только RAG без сравнения с plain-ответом

```bash
python main.py --lm-studio-url http://localhost:1234 --no-plain
```

### С режимом "не знаю"

```bash
python main.py --lm-studio-url http://localhost:1234 --dont-know-threshold 5.0
```


### Аргументы командной строки

| Аргумент | По умолчанию | Описание |
|---|---|---|
| `--index` | `../day21/index/structural.json` | Путь к индексу |

| `--questions` | `questions.json` | Путь к файлу с вопросами |
| `--lm-studio-url` | `http://localhost:1234` | URL LM Studio API |
| `--model` | `qwen/qwen3-5b` | Модель в LM Studio |
| `--top-k` | `20` | Кол-во кандидатов для cosine поиска |
| `--rerank-top-k` | `5` | Кол-во чанков после реранкинга |
| `--window` | `3` | Соседних чанков с каждой стороны (sentence-window) |
| `--dont-know-threshold` | `None` | Порог rerank score для режима "не знаю" (например `5.0`) |
| `--embed-model` | `nomic-ai/nomic-embed-text-v1.5-GGUF` | LM Studio модель для эмбеддингов |
| `--output` | `results.json` | Куда сохранить результаты |
| `--no-plain` | `False` | Не запускать режим без RAG |


## Формат ответа

Модель обязана вернуть валидный JSON:


```json
{
  "answer": "Хозяина Бима звали Иван Иваныч.",
  "sources": [
    "bim.md / # 1. ДВОЕ В ОДНОЙ КОМНАТЕ (part 24) (chunk: bim.md::structural::25)"
  ],
  "quotes": [
    "Ты молодец, Бим! Будем жить вместе, хотя бы и без родословной."
  ]
}
```

| Поле | Описание |
|---|---|
| `answer` | Развёрнутый ответ на русском языке |
| `sources` | Список источников с именем файла, разделом и chunk_id |
| `quotes` | 1–3 дословные цитаты из контекста подтверждающие ответ |

Если ответа в контексте нет — модель возвращает `"answer": "НЕ ЗНАЮ"` с пустыми `sources` и `quotes`.

## Режим "не знаю"

Если задан `--dont-know-threshold`, система проверяет лучший rerank score после реранкинга. Если он ниже порога — запрос к модели не отправляется, ответ формируется автоматически:

```
⚠️  НЕ ЗНАЮ: Не могу ответить на этот вопрос на основе имеющихся документов.
             Пожалуйста, уточните вопрос или обратитесь к другому источнику.
```

Рекомендуемые значения порога: `5.0`–`7.0`. При значении `None` (по умолчанию) режим отключён.

## LLM реранкер

Для каждого кандидата из top-K модель оценивает релевантность к вопросу по шкале 0–10:

- `0` — текст не связан с вопросом
- `5` — текст частично связан
- `10` — текст прямо отвечает на вопрос

Реранкер делает `--top-k` запросов к API на каждый вопрос. При `--top-k 20` это 20 запросов перед основным. Для ускорения уменьши `--top-k`:

```bash
python main.py --top-k 10 --lm-studio-url http://localhost:1234
```

```json
[
  {
    "id": 3,
    "question": "как звали хозяина бима чёрное ухо",
    "expected": "Иван Иваныч",
    "expected_source": "bim.md / 1. ДВОЕ В ОДНОЙ КОМНАТЕ",
    "answer_plain": "Ответ модели без RAG...",
    "answer_rag": "Хозяина Бима звали Иван Иваныч.",

    "sources": [
      "bim.md / # 1. ДВОЕ В ОДНОЙ КОМНАТЕ (part 24) (chunk: bim.md::structural::25)"
    ],
    "quotes": [
      "Ты молодец, Бим! Будем жить вместе, хотя бы и без родословной."
    ],
    "dont_know": false,
    "chunks": [
      {
        "source": "bim.md",
        "section": "# 1. ДВОЕ В ОДНОЙ КОМНАТЕ (part 24)",
        "cosine_score": 0.827,
        "rerank_score": 10.0
      }
    ],
    "checks": {
      "has_answer": true,
      "has_sources": true,
      "has_quotes": true,
      "dont_know": false
    }
  }
]
```
