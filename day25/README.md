# RAG Chat — День 25

Мини-чат CLI с RAG-поиском, историей диалога и памятью задачи.

## Структура проекта

```
rag_chat/
├── main.py          # Точка входа, CLI-интерфейс
├── chat.py          # Движок чата: история, RAG, промпт, LLM
├── vector_store.py  # Загрузка индекса, cosine similarity, поиск
├── task_memory.py   # Управление памятью задачи (task_state)
└── index/
    └── structural.json   # Индекс с эмбеддингами (подготовьте заранее)
```


## Установка зависимостей

Используются только стандартные библиотеки Python (3.10+).  
Внешние зависимости: отсутствуют.

## Переменные окружения


```bash
export OPENAI_API_URL="https://openai.api.proxyapi.ru/v1"
export OPENAI_API_KEY="sk-..."
```

## Запуск

```bash
python main.py --model gpt-4o-mini
```

### Все параметры

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--model` | *обязательный* | Модель OpenAI API |
| `--ollama-host` | `http://172.27.112.1:11435` | Хост Ollama для эмбеддингов |
| `--index` | `index/structural.json` | Путь к файлу индекса |
| `--top-k` | `5` | Кол-во чанков при RAG-поиске |

## Команды в чате

| Команда | Действие |
|---|---|
| `/help` | Список команд |
| `/memory` | Показать текущую память задачи |
| `/sources` | Источники последнего ответа |
| `/reset` | Сбросить историю и память задачи |
| `/quit`, `/exit` | Выйти |


## Архитектура

### Как работает каждый запрос


```
Сообщение пользователя
        │
        ▼
task_memory.build_rag_query()
  (цель + ограничения + сообщение)
        │
        ▼
vector_store.search()  ──→  Ollama embeddings  ──→  cosine similarity
        │
        ▼
Системный промпт:

  SYSTEM_BASE + task_state + найденные чанки
        │
        ▼
LLM API (история диалога + промпт)
        │
        ▼
Ответ + источники  ──→  task_memory.update()
```


### Task State (память задачи)


Структура, которая накапливается по ходу диалога:

```json
{
  "goal": "цель пользователя",
  "clarified": ["факт 1", "факт 2"],
  "constraints": ["ограничение 1"],
  "open_questions": ["вопрос 1"]
}
```

- **goal** — общая цель диалога, формулируется LLM из первых сообщений
- **clarified** — что пользователь уточнил или подтвердил (только накапливается)
- **constraints** — ограничения, требования, зафиксированные термины (только накапливается)

- **open_questions** — вопросы, которые ещё не прояснены

### RAG-запрос с учётом контекста

Вместо поиска только по последнему сообщению используется расширенный запрос:

```
rag_query = goal + constraints + user_message
```

Это позволяет корректно находить контекст для уточняющих вопросов вида
"а если без библиотек?" — goal и constraints добавляют необходимый контекст.

## Формат индекса


Файл `index/structural.json` — список объектов:

```json
[
  {
    "chunk_id": "bim.md::structural::0",
    "text": "...",
    "source": "bim.md",
    "title": "bim",
    "section": " (part 1)",
    "embedding": [0.123, ...]
  }
]
```

Модель эмбеддингов определяется из поля `embedding_model` первого чанка
(если отсутствует — используется `nomic-embed-text`).
