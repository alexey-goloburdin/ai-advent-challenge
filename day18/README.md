# Day 18 — MCP Reminder Agent

ИИ-агент с естественным языком для управления напоминаниями.
Сервер и агент — два независимых процесса, общаются по SSE (HTTP).

## Структура проекта

```
.
├── reminder_server.py   # MCP-сервер (SSE транспорт, запускается отдельно)
├── reminder_agent.py    # Агент-клиент (24/7, коннектится к серверу по URL)
├── reminders.json       # База данных (создаётся автоматически)
└── README.md
```

## Установка

```bash
pip install "mcp[cli]" uvicorn
```


Требуется Python 3.10+.

## Запуск


Терминал 1 — сервер:
```bash
python reminder_server.py

# Сервер запущен на http://127.0.0.1:8000/sse
```

Терминал 2 — агент:
```bash
export OPENAI_BASE_URL=https://api.openai.com/v1
export OPENAI_API_KEY=sk-...
python reminder_agent.py
```

Остановка: `Ctrl+C` в каждом терминале независимо.

```

─────────────────────────────  ──────────────────────────────────────
reminder_server.py             reminder_agent.py

│                              │
│  FastMCP (SSE)               │  sse_client("http://...../sse")
│  tools:                      │        │
│  • add_reminder()     ◄──────┼────────┤  chat_loop()
│  • get_due_reminders()◄──────┼────────┤    └── chat_turn()
│  • list_reminders()   ◄──────┼────────┤          └── LLM (urllib)

│                              │        │
│  reminders.json              │        └── polling_loop()
│  (персистентное хранение)    │              └── get_due_reminders()
```


Сервер и агент — **независимые процессы**. Сервер можно перезапустить без остановки агента, и наоборот.

## Почему SSE, а не stdio

stdio транспорт в MCP по спецификации предполагает что клиент **спавнит** сервер как subprocess и общается через пайп. Два независимых процесса через stdio невозможны — у каждого терминала свой stdin/stdout.

SSE (HTTP) — правильный транспорт когда сервер и клиент должны жить независимо.

## MCP Tools сервера

| Tool                  | Описание                                              |
|-----------------------|-------------------------------------------------------|
| `add_reminder`        | Добавить напоминание (`text`, `delay_seconds`)        |

| `get_due_reminders`   | Вернуть сработавшие напоминания, пометить как `done`  |

| `list_reminders`      | Вернуть все напоминания                               |


## Формат хранилища

`reminders.json` — JSON-массив, создаётся автоматически:


```json
[

  {

    "id": 1,
    "text": "Сделать кофе",
    "fire_at": 1700000300.0,

    "fire_at_human": "2024-11-14 12:05:00",
    "done": true,
    "created_at": 1700000000.0
  }
]
```

Данные сохраняются между перезапусками — невыполненные напоминания не теряются.
