# FlowCast — Техническая документация

## API

Базовый URL: https://api.flowcast.io/v1

Аутентификация: Bearer-токен в заголовке Authorization.
Токен генерируется в разделе «Настройки» → «API-ключи». Каждый ключ можно ограничить
по правам (read-only, read-write) и привязать к конкретному подкасту.

### Основные эндпоинты

**Подкасты**
- GET /podcasts — список всех подкастов аккаунта
- GET /podcasts/{id} — данные конкретного подкаста
- PATCH /podcasts/{id} — обновить метаданные подкаста

**Эпизоды**
- GET /podcasts/{id}/episodes — список эпизодов
- GET /episodes/{id} — данные эпизода
- POST /episodes — создать черновик эпизода
- PATCH /episodes/{id} — обновить эпизод
- POST /episodes/{id}/publish — опубликовать немедленно
- POST /episodes/{id}/schedule — запланировать публикацию (параметр: publish_at в ISO 8601)
- DELETE /episodes/{id} — удалить черновик (опубликованные эпизоды не удаляются через API)

**Загрузка файлов**
- POST /uploads/request — получить presigned URL для загрузки напрямую в S3
- GET /uploads/{upload_id}/status — статус обработки загруженного файла

Загрузка файлов всегда двухэтапная: сначала получаете presigned URL, затем загружаете
файл напрямую через PUT-запрос на этот URL. Это позволяет загружать файлы до 2 ГБ
без проксирования через серверы FlowCast.

**Аналитика**
- GET /analytics/episodes/{id} — статистика по эпизоду (параметры: from, to в формате YYYY-MM-DD)
- GET /analytics/podcast/{id}/overview — сводная статистика подкаста

**Транскрипции**
- POST /episodes/{id}/transcribe — запустить транскрипцию
- GET /episodes/{id}/transcription — получить результат (статус: pending, processing, done, failed)

### Лимиты API

| План       | Запросов/час | Запросов/сутки |
|------------|-------------|----------------|
| Free       | 60          | 500            |
| Creator    | 600         | 10 000         |
| Studio     | 2 000       | 50 000         |
| Enterprise | Без лимита  | Без лимита     |

При превышении лимита возвращается 429 Too Many Requests с заголовком
Retry-After, указывающим когда можно повторить запрос.

### Коды ошибок

- 400 Bad Request — некорректные параметры, проверьте тело запроса
- 401 Unauthorized — API-ключ отсутствует или невалиден
- 403 Forbidden — действие не разрешено для данного ключа или плана
- 404 Not Found — ресурс не существует или не принадлежит вашему аккаунту
- 409 Conflict — конфликт состояния (например, попытка опубликовать уже опубликованный эпизод)
- 413 Payload Too Large — файл превышает допустимый размер
- 422 Unprocessable Entity — файл загружен, но не может быть обработан (формат не поддерживается или файл повреждён)
- 429 Too Many Requests — превышен лимит запросов
- 500 Internal Server Error — ошибка на стороне FlowCast, повторите позже или обратитесь в поддержку

## Webhooks

FlowCast может отправлять уведомления на ваш сервер при наступлении событий.
Настройка: «Настройки» → «Webhooks» → добавить URL и выбрать события.

Поддерживаемые события:
- episode.published — эпизод опубликован
- episode.scheduled — эпизод поставлен в очередь
- transcription.completed — транскрипция готова
- transcription.failed — транскрипция завершилась с ошибкой
- upload.processed — загруженный файл обработан
- subscription.created — новый платный подписчик
- subscription.cancelled — подписчик отписался

Webhook-запросы подписываются HMAC-SHA256, ключ подписи доступен в настройках.
Ваш сервер должен вернуть 200 OK в течение 10 секунд. При неудаче FlowCast
повторит попытку через 1, 5, 30, 60 и 120 минут.

## RSS-фид

RSS-фид подкаста доступен по адресу: https://feed.flowcast.io/{podcast_slug}

Фид соответствует спецификации Apple Podcasts RSS и содержит все стандартные
теги: itunes:author, itunes:category, itunes:explicit, itunes:image и т.д.

При использовании кастомного домена фид также доступен по https://podcast.yourdomain.com/feed.

Теги главы (chapters) в формате Podlove Simple Chapters поддерживаются —
добавьте главы при редактировании эпизода.

## Транскрипция

Транскрипция выполняется асинхронно. Примерное время: 1 минута аудио → 1–2 минуты обработки.

Результат доступен в трёх форматах через API:
- text — чистый текст
- srt — субтитры с таймкодами
- json — полная разметка с таймкодами для каждого слова и определением спикеров

Поддерживаемые языки: русский, английский. Смешанный язык (code-switching)
определяется автоматически, но точность ниже.

Квота транскрипции считается по минутам аудио. Неиспользованные минуты
не переносятся на следующий месяц.

## Обработка аудио

После загрузки файл проходит автоматическую обработку:
1. Конвертация в WAV 48 кГц для внутреннего хранения
2. Нормализация громкости до -16 LUFS (стандарт для подкастов)
3. Генерация MP3 в запрошенном битрейте для публикации
4. Создание waveform-данных для визуализации в редакторе

Обработка занимает от 30 секунд до 5 минут в зависимости от длины файла.
Статус отслеживайте через GET /uploads/{upload_id}/status или webhook upload.processed.

## Хранилище

Все файлы хранятся в облачном хранилище с троекратным резервированием.
Оригинальный файл (в исходном формате) хранится отдельно от обработанных версий.

При удалении эпизода из интерфейса файлы помечаются как удалённые и физически
удаляются через 30 дней — это даёт возможность восстановления.

На плане Free хранилище не освобождается автоматически при удалении эпизодов
сразу — нужно подтвердить «Очистить корзину» в разделе «Хранилище».

## Встроенный плеер

Для встраивания плеера на сайт используйте iframe:

  <iframe src="https://embed.flowcast.io/e/{episode_id}" width="100%" height="180" frameborder="0"></iframe>

Параметры iframe:
- theme=dark / theme=light — тема плеера
- autoplay=1 — автовоспроизведение (блокируется большинством браузеров без взаимодействия)
- start=120 — начать воспроизведение с указанной секунды

Плеер адаптивный, минимальная ширина — 300 px.
