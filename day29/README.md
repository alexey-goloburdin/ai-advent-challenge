# bench.py — оптимизация локальной LLM для извлечения реквизитов

Скрипт для задания **День 29: Оптимизация локальной LLM**.

Сравнивает два профиля работы модели на задаче извлечения реквизитов
российских юридических лиц из неструктурированного текста.


---

## Что сравнивается

| Параметр | `before` | `after` |
|---|---|---|
| `temperature` | 0.7 | 0.1 |
| `max_tokens` | 2048 | 512 |
| `top_k` | — | 20 |
| `presence_penalty` | — | 1.5 |
| `reasoning` | `"off"` (оба) | `"off"` (оба) |
| Системный промпт | минимальный | структурированный |
| Формат ответа | произвольный | строгий JSON |

---


## API

Скрипт использует **нативный LM Studio REST API**: `POST /api/v1/chat`


В отличие от OpenAI-совместимого `/v1/chat/completions`, нативный API поддерживает:
- `"reasoning": "off"` — прямое отключение reasoning/thinking
- `"store": false` — stateless режим (история не сохраняется в LM Studio)
- поле `input` вместо `messages`
- ответ в `output[].content` вместо `choices[0].message.content`

Параметр `"reasoning": "off"` применяется ко **всем** запросам, включая профиль `before`.

Зависимостей нет — используется только стандартная библиотека Python 3.8+.

```bash
python bench.py --help
```

## Запуск

### Сравнить оба профиля (основной сценарий)

```bash
python bench.py --url http://192.168.56.1:1234
```

### Только оптимизированный профиль

```bash
python bench.py --url http://192.168.56.1:1234 --profile after
```

### Несколько прогонов для усреднения

```bash
python bench.py --url http://192.168.56.1:1234 --runs 3
```

### Другая модель

```bash
python bench.py --url http://192.168.56.1:1234 --model qwen/qwen3.5-35b-a3b
```

### Подробный вывод с ответами модели

```bash
python bench.py --url http://192.168.56.1:1234 --verbose
```

### Все параметры вместе

```bash
python bench.py \
  --url http://192.168.56.1:1234 \
  --model qwen/qwen3.5-9b \
  --profile both \
  --runs 3 \
  --timeout 180 \
  --verbose
```


---

## Параметры CLI

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--url` | *(обязательный)* | Базовый URL LM Studio |
| `--model` | `qwen/qwen3.5-9b` | Идентификатор модели |
| `--profile` | `both` | `before` / `after` / `both` |

| `--runs` | `1` | Прогонов на кейс (для усреднения) |
| `--timeout` | `120` | Таймаут запроса в секундах |
| `--verbose` / `-v` | `false` | Печатать ответы модели |

---

## Тест-кейсы

| ID | Описание |
|---|---|
| `simple` | Минимальный набор реквизитов в одну строку |
| `invoice` | Реквизиты в формате счёта/договора |
| `noisy` | ИП среди лишнего текста (договор) |
| `missing_fields` | Неполные данные — часть полей отсутствует |

## Структура извлекаемых данных

```python

@dataclass
class Bank:
    name: str             # наименование банка
    BIC: str              # БИК

    current_account: str  # расчётный счёт
    corporate_account: str  # корреспондентский счёт

@dataclass
class Customer:
    name: str       # полное название юридического лица
    INN: str        # ИНН
    OGRN: str       # ОГРН или ОГРНИП
    address: str    # юридический адрес
    signatory: str  # подписант
    bank: Bank      # банковские реквизиты
```


---

## Метрики оценки

Каждый кейс получает оценку **0–3**:

- **+1** — ответ модели является валидным JSON
- **+1** — ИНН совпадает с ожидаемым значением
- **+1** — в ответе присутствуют банковские реквизиты (поле `BIC`)

В итоговой таблице также выводится:

- среднее время ответа (секунды)
- количество completion-токенов
- скорость генерации (tokens/sec)
- сравнение скорости between профилями
